# 模块地图自检清单

交付 `docs/module-map.md` 前使用本清单自检。

## 必须通过

- 文档是导航地图，不是完整模块手册。
- 顶层地图只列稳定业务域和关键支撑模块，不列每个文件夹。
- 每个模块都有路径、一句话职责和有用的下一步指针。
- 只有影响架构或 AI 行为的跨模块硬边界才写入地图。
- 普通编码规范不在地图中展开，而是链接到架构或规范文档。
- 重要隐性业务勾连已作为少量高风险摘要写入地图，或链接到影响地图。
- 禁区或生成区已显式列出；如果没有，章节中写“无”。
- 不包含详细实现、函数、类、字段和代码片段。
- ADR 的理由只链接，不复制到地图里。
- 最终文档足够短，适合编码任务开始前阅读。

## 压缩规则

- 如果某个模块条目超过 3 行，创建或链接模块子文档。
- 如果顶层模块超过约 15 项，按业务域或支撑域重新分组。
- 如果某段在解释历史或替代方案，把它移到 ADR。
- 如果某段在解释业务行为，把它移到规约。
- 如果某段在表达“改 A 还要看 B”，把完整规则移到影响地图，模块地图只保留最高风险摘要或链接。

## 失败信号

- 地图读起来像 README。
- 地图照搬了文件系统。
- 地图重复了语义搜索本该解决的内容。
- 地图包含过期 TODO 或未来计划。
- 地图中存在无法防止 agent 具体错误的行。
