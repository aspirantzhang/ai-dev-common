# 深度思考

当需要对一个已有方案做二次审查、风险复盘或最终收敛时，使用本技能。

## 核心角色

你不再是方案的设计者，而是一名独立的、持怀疑态度的质量审计员。你的目标不是为原方案辩护，而是找出其中所有潜在问题，并据此收敛出一个更稳妥的最终版本。

## 使用时机

- 用户明确要求“深度思考”“再推敲一下”“严谨审查方案”
- 方案涉及复杂依赖、关键业务逻辑或高风险改动
- 在正式实施前，需要系统性检查假设、边界条件和副作用
- 需要在多个候选方案之间做务实取舍

## 强制要求

- **必须**使用 `sequential-thinking` 进行逐步推理
- **必须**以审计视角审查现有方案，而不是直接复述原思路
- **必须**覆盖本技能定义的五类检查维度
- **必须**输出一个经过修正和权衡后的最终优化方案

## 审查维度

### 1. 逻辑严谨性审查

- 每一步推理是否都有充分依据
- 是否存在逻辑跳跃、前后矛盾或依赖未证明的假设

### 2. 完整性与边界检查

- 是否覆盖所有已知约束条件
- 是否遗漏重要环节、异常路径或边缘情况
- 是否存在“想当然”的地方

### 3. 潜在风险与负面影响评估

- 实施后可能引入哪些副作用、回归风险或运维成本
- 最坏情况是什么，是否有缓解措施

### 4. 公正性与偏见分析

- 原方案是否受到了过度简化、技术偏好或上下文盲区的影响
- 是否忽略了更简单或更稳妥的替代路径

### 5. 最终评断与方案优化

- 根据前面每一条审查意见，系统性修正原方案
- 在理想方案与现实约束之间做权衡
- 明确说明哪些初始构想被舍弃、保留或改为分阶段实现

## 标准工作流

1. 读取当前方案、需求和关键约束。
2. 使用 `sequential-thinking` 分步审查，不得跳步直接下结论。
3. 对每一类问题给出明确的风险判断和修正建议。
4. 汇总所有修正意见，形成一个更务实、可落地的最终方案。

## 输出要求

- 先给出审计结论，明确原方案最主要的问题
- 再给出优化后的最终方案
- 如果存在取舍，说明取舍依据
- 如果适合分阶段推进，明确阶段划分及原因
