# 影响地图模板

创建或重写 `docs/impact-map.md` 时使用本模板。影响地图应保持轻量，默认使用分节式 Markdown 条目，不使用表格，也不建立复杂编号或状态体系。

```markdown
# 跨模块影响地图

> 本文档记录“改 A 可能暗伤 B”的非显而易见跨模块影响。它不是依赖图，也不是完整测试清单。
> 修改某个模块前，先按“触发”列查找相关条目，并把“应对”列纳入提案、规约、设计或测试范围。

## 影响项

### 修改订单取消流程

- **触发**: 修改订单取消、退款前置条件或取消事件语义。
- **影响**: Billing、Inventory、Notification。
- **机制**: 取消订单会触发退款、库存释放和用户通知，但这些影响不一定通过直接 import 暴露。
- **应对**:
  - 检查退款、库存释放、取消通知是否仍符合业务预期。
  - 补充或运行覆盖取消流程的跨模块测试。
- **依据**: 历史回归 / 人工确认。

### 新增订单状态

- **触发**: 新增、重命名或合并订单状态。
- **影响**: Analytics、BI 报表。
- **机制**: 报表维护独立状态映射，类型检查无法发现映射遗漏。
- **应对**:
  - 更新状态映射。
  - 确认漏斗和运营报表口径。
  - 必要时通知运营或 BI。
- **依据**: issue #1456。

### 修改金额精度或货币单位

- **触发**: 修改金额精度、货币单位或结算金额语义。
- **影响**: Notification、Finance tools。
- **机制**: 下游模板和对账脚本可能维护独立格式化逻辑。
- **应对**:
  - 检查金额格式化、对账脚本和通知模板。
  - 运行覆盖结算通知或对账的跨模块测试。
- **依据**: PR #1842。

## 维护规则

- 只记录非显而易见、跨模块、会造成真实后果的影响。
- 能被类型系统、linter、依赖图稳定发现的问题，不写入本地图。
- 没有线索或人类确认的猜测，不写入正式条目。
- 条目超过约 15 条时，按触发模块分组。
- 影响已被测试或工具覆盖时，在“依据”中短标注“已缓解”，不要展开长历史。
```

可以根据项目调整字段名，但必须保留核心问题：触发什么、影响谁、为什么、怎么检查。
