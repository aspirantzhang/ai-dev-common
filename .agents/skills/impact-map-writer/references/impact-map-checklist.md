# 影响地图自检清单

交付 `docs/impact-map.md` 前使用本清单自检。

## 必须通过

- 每条都在回答“改 A 会不会无意中影响 B”。
- 每条都有触发、影响、机制、应对。
- 应对动作具体，Agent 看完知道要检查什么。
- 条目不是普通 import、显式调用或同模块内部联动。
- 条目不是纯猜测，至少有历史、代码线索、issue、PR、规约、测试或人类确认。
- 文档没有重复模块地图、ADR、依赖规则或测试清单。

## 不应写入

- 类型系统、编译器、linter、外键约束能稳定发现的问题。
- 显而易见的“改接口要检查调用方”。
- 只读数据关系，没有状态变化、业务语义变化或副作用。
- 高频到每次都触发的耦合；这种应优先重构或自动化检查。
- 没有具体应对动作的提醒。

## 压缩规则

- 一条影响项过长时，压缩字段内容，把背景移到 ADR、issue 或 post-mortem。
- 条目超过约 15 条时，按触发模块分组。
- 影响已被自动化检查覆盖时，保留短标注即可，不展开历史。
