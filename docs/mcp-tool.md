## MCP 工具使用时机与规则

### dbhub - mysql

- **用途**：查询数据库表结构
- **适用场景**：数据库设计编写
- **使用时机**：需要了解表结构时

### sequential-thinking

- **用途**：复杂问题的逐步分析
- **适用场景**：需求分析、方案设计、问题排查
- **使用时机**：遇到复杂逻辑或多步骤问题时

### context7

- **用途**：查询最新的技术文档、API 参考和代码示例
- **适用场景**：技术调研、最佳实践获取
- **使用时机**：需要了解新技术或验证实现方案时
- **使用规范**：`resolve-library-id → get-library-docs → 抽取关键段落`

### desktop-commander

- **用途**：文件读取、代码读取、文本搜索
- **适用场景**：需求分析、方案设计、问题排查
- **使用时机**：文件读取，代码读取，文件搜索，代码修改，文本编辑，大文件操作，基于模式的替换，对文件进行重写或改写

### fast-context

- **用途**：本地代码库的语义检索与快速定位
- **适用场景**：大型项目导航、跨文件逻辑关联分析、快速定位功能实现位置
- **使用时机**：当不知道具体文件名但知道功能关键词时；需要从全局视角理解代码逻辑或查找模糊匹配的实现细节时
- **使用规范**：语义搜索 `fast_context_search` → 返回文件+行号范围+grep关键词建议 → `grep` 二次精确搜索 → 定位目标代码
