# 通用字段规则

- 默认可空性：若未特别澄清，即可认为是选填，即可接收 null
- 布尔类型字段：数据库可以存储为 1/0，但前后端接口交互时必须用 true/false 而不是数字。
  - 输入校验示例：`z.boolean()`
  - 输出 DTO 示例：`is_active: Boolean(record.is_active)`
- 必填与条件逻辑：当实体要求是必填，但其规则不需要接收/存储该实体时，不应该要求其为必填
- Decimal 类型字段：
  - 普通比例、系数、分值可使用 number。
  - 薪资、税费、社保、公积金、奖金等金额类字段，优先使用“分”为单位的 integer 或 decimal string，避免 JavaScript 浮点精度问题。
  - 金额展示格式由前端或 formatter 统一处理，业务计算不要依赖展示字符串。
- 字典值字段：在输出类型的接口中，都需要额外加入 `${原字段名}_name` 的字段，其值为字典中的名称
- 日期时间类字段：使用标准格式 `YYYY-MM-DD HH:mm:ss`；跨时区场景必须在字段说明中明确时区来源。
